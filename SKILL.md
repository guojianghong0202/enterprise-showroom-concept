# 企业品牌展厅概念策划

把项目当作企业事实、品牌叙事和空间体验的联合策划任务。先核实事实并让用户选择故事线，再展开完整方案；不要从零散资料直接跳到最终提案。

## 核心规则

1. **严格执行两阶段确认。** 第一阶段输出需求诊断和 2–3 条故事线后停止。只有用户明确选择、组合或批准故事线，才进入第二阶段。
2. **区分事实和建议。** 企业事实必须关联证据编号；策划、视觉和互动建议必须标明为建议。
3. **只补充官方来源。** 允许企业官网、官方公众号、年报、政府和行业协会官方资料。第三方文章不能替代企业事实来源。
4. **资料不足就降级。** 缺少尺寸、出入口或清晰图纸时，输出关系、原则和待确认项，不伪造具体面积或正式动线。
5. **保持概念边界。** 使用“概念规划建议”“概念效果参考”或“提案视觉”。不输出施工图、精确预算或工程结论。
6. **不得覆盖原成果。** 目标文件存在时生成版本文件，或先取得用户明确授权。

## 开始前

读取用户提供的品牌文件、空间图纸、现场照片和官方入口。只询问缺失且会改变方案的问题，避免重复提问。

读取 `references/input-and-evidence-schema.md`，建立项目清单和证据台账。将企业事实标为：`已核实`、`用户提供待核实`、`来源冲突` 或 `缺少来源`。

运行输入检查：

```powershell
py -3 scripts/validate_input_manifest.py <manifest.json> --stage phase1
```

存在 `BLOCKER` 时先解决或向用户说明；只有 `WARNING` 时按相应降级方式继续。

## 第一阶段：诊断与故事线比选

1. 汇总项目目标、核心受众、接待场景、参观任务和必展内容。
2. 识别品牌事实、空间约束、资料缺口和来源冲突。
3. 仅在必要时按照 `references/official-source-policy.md` 查询官方来源。
4. 读取 `references/enterprise-narrative-frameworks.md`，生成 2–3 条战略角度不同的故事线。
5. 每条故事线包含主题、核心主张、叙事结构、情绪曲线、展区映射、视觉关键词、优势、风险和适用条件。
6. 使用质量量表评分，输出 `01_需求诊断与证据台账.md` 与 `02_故事线比选与确认记录.md`。
7. 运行第一阶段交付检查，然后停止并等待用户确认。

```powershell
py -3 scripts/validate_concept_package.py <成果目录> --stage phase1
```

不要在同一轮继续生成完整母稿。用户提出融合方案时，记录融合后的故事线编号和确认内容。

## 第二阶段：完整概念方案

确认 `storyline.confirmed=true` 且存在 `selected_id` 后：

1. 读取 `references/zoning-and-visitor-flow.md`，把故事线映射为展区、内容节奏和参观动线。
2. 为每个展区建立内容卡：目标、核心信息、事实证据、展示内容、展项、互动、视觉、提示词和待确认项。
3. 读取 `references/visual-direction-system.md`，形成色彩、材质、灯光、图形和多媒体方向。
4. 按 `references/render-prompt-handoff.md` 生成效果图交接卡。
5. 如果 `advertising-render-prompts` 可用，调用它深化模型提示词；否则使用本 Skill 的通用提示词模板。不要自动安装依赖。
6. 按 `references/output-contract.md` 和 `assets/` 模板生成六项交付文件。
7. 使用 `references/quality-rubric.md` 做内容评分和阻断检查。
8. 运行第二阶段输入与交付验证。

```powershell
py -3 scripts/validate_input_manifest.py <manifest.json> --stage phase2
py -3 scripts/validate_concept_package.py <成果目录> --stage phase2
```

存在 `BLOCKER` 时不得标记为“概念提案可用”。

## 默认交付

用户未指定目录时，写入当前项目的 `成果/<项目名称>/`：

```text
01_需求诊断与证据台账.md
02_故事线比选与确认记录.md
03_企业展厅概念策划母稿.md
04_PPT逐页提案脚本.md
05_分区效果图提示词.md
06_来源与待确认清单.md
```

用户只要求对话输出时，不强制创建文件。

## 失败与越界处理

- 官方来源无法访问：记录链接、失败原因和替代获取建议，不推测正文。
- 官方来源相互矛盾：并列记录证据，写入待核实问题。
- 图纸或照片无法辨认：请求原始文件，不凭模糊图判断。
- 用户要求施工图、正式预算或专业工程判断：说明本 Skill 的概念策划边界，并列出所需专业输入。
- 效果图 Skill 不可用：继续主体策划，只使用内置提示词模板。

最终说明已生成内容、采用的假设、验证结果、待确认问题和成果路径。
