# 展项、互动与运营设计

## 从认知变化开始

每个展项 `X##` 先回答三件事：访客原来怎么想、需要理解或相信什么、体验后产生什么变化。不能把“做一块大屏”“增加互动”当目标。

## 展项卡

- 编号、所属 `Z##`、目标受众与决策目标。
- 核心内容、`[E###]` 企业证据和版权状态。
- 载体：实物、模型、图文、机械、数字、影音或沉浸。
- 访客动作、系统反馈和预期认知变化。
- 单人/多人方式、建议时长、讲解依赖和排队风险。
- 设备故障或无电时的低技术替代。
- 内容更新、维护、人员操作、清洁、能耗和停机风险。
- 包容性提醒、专业复核事项和适用预算档位。

## 载体选择

| 内容任务 | 优先载体 | 谨慎使用 |
| --- | --- | --- |
| 证明真实能力 | 实物、样件、剖切模型、可追溯数据 | 只有宣传片 |
| 解释复杂原理 | 分层模型、流程动画、机械演示 | 长篇墙文 |
| 比较方案价值 | 案例前后对照、场景推演、数据图 | 无结论的炫技互动 |
| 表达历程与人物 | 实物、档案、口述、时间关系 | 纯装饰复古符号 |
| 表达生态网络 | 地图、关系图、可更新名录 | 一次性固定名单 |

数字媒体只在内容需要动态、选择、模拟或更新时使用。

## 互动闭环

```text
吸引 → 看懂动作 → 操作 → 获得反馈 → 形成结论 → 回到故事线
```

检查启动提示、完成时间、多人观看、错误恢复、排队、无障碍替代和退出方式。互动结果应能用一句话解释，不依赖讲解员补救。

## 低技术替代

每项数字或机械展项必须给出静态/低技术方案，例如：

- 触控检索 → 分层索引图文与二维码后续内容。
- 沉浸影片 → 精简主视觉、字幕摘要和关键帧故事板。
- 动态模型 → 剖切模型、透明结构或步骤翻页。
- 实时数据 → 最近核验数据快照与更新时间。

替代方案不是“效果差版本”，而是保证信息目标在故障、预算收缩或低感官需求下仍成立。

## 运维检查

记录内容责任人、更新频率、素材格式、审核链、设备重启、备件、人员培训和退出机制。高维护展项必须说明它提供的独特价值及降级路径。
