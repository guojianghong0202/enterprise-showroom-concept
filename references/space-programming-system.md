# 专业空间规划系统

## 顺序

先确定信息与故事，再做展区关系；先做关系与区间，再做概念落位。不要按墙面数量平均分内容。

```text
访客任务 → 故事节点 → 展区计划 → 邻接图 → 路线 → 面积/停留区间 → 风险复核
```

## 展区计划 `Z##`

每个展区至少记录：名称、叙事角色、优先级、目标受众、关键信息、证据、面积比例区间、前置/后继、必须邻接、建议停留、展项、视觉、导视、效果图交接、假设和复核事项。

面积表达：

- 图纸充分且功能稳定：可用精确比例，总和 `100% ±1%`。
- 方案早期或资料不足：用比例区间，需满足 `sum(min) <= 100 <= sum(max)`。
- 不把比例换算为精确平方米，除非有效面积口径已确认。

## 邻接关系

为每个 `Z##` 记录：

- `must_follow`：叙事或运营上必须在前一节点之后。
- `adjacent_to`：必须直接或近距离相邻。
- `avoid_adjacent_to`：声音、光线、保密或拥堵冲突。
- `optional`：时间不足时可跳过。

邻接图必须连通。相邻不是“都在主路线”同义词；要说明内容、运营或空间原因。

## 路线 `R##`

支持：

- 主讲解路线：覆盖总主张、关键能力和出口行动。
- 自由参观路线：强化章节识别、当前位置和可跳转性。
- 重点客户路线：在限定时间内优先访问决策相关节点。

每条路线包含入口、出口、展区序列、可跳过节点、停留区间、容量风险和讲解依赖。已知入口/出口时必须对应；未知时标为候选，不能声明确定路线。

机器字段固定为：

```text
space_program: allocation_mode, placement_status
routes[]: id, name, audience_ids, zone_ids, start, end, duration_minutes{min,max}
zones[]: id, name, priority, area_pct 或 area_range_pct{min,max},
         adjacent_to, exhibit_ids, visual_direction, wayfinding,
         render_handoff, open_questions
```

`duration_minutes` 必须是对象，不能写成数组或文本。路线存在 `audience_ids` 时优先使用对应受众的 `expected_visit_minutes`；没有受众级目标时才回退项目总体区间。

## 节奏节点

至少检查：

1. 入口识别：快速回答“是谁、为什么值得看”。
2. 首个记忆点：在信息过载前建立核心主张。
3. 核心高潮：最有说服力的证据或体验。
4. 回望节点：帮助访客整合前文，而非继续加信息。
5. 出口行动：总结、洽谈、招商、合作或传播承接。

路线各展区停留区间应覆盖预计总时长；不覆盖时给出调整警告。

## 置信度与降级

- `high`：图纸、尺寸、出入口和主要限制可读，可做概念落位。
- `medium`：部分信息可用，只对可靠区域落位，其余保留区间。
- `low`：只做邻接、面积原则和候选路线。
- `unknown`：先建立资料清单，不输出空间确定性判断。

消防、结构、承重、机电、通行净宽、设备安装和法规符合性始终列入专业复核。
