# 效果图提示词交接

## 可选调用

始终先生成模型无关的交接卡。只有用户明确要求深化效果图提示词或生成概念视觉时，才检查并调用 `advertising-render-prompts`；实际出图仍需用户明确请求。不存在时使用本文件模板，不自动安装依赖，也不中断策划交付。

## 标准交接卡

```text
展区编号与名称：
对应故事/路线/展项/PPT：S## / R## / X## / P##
图像用途：概念效果参考 / 提案视觉 / 风格探索
目标模型：已知模型或“通用”
空间条件：面积、层高、入口、结构、视线和不可改造条件
故事线角色：该展区在整体叙事中的作用
核心内容：事实摘要及证据编号
访客与情绪目标：
空间构成：墙面、展台、展柜、屏幕、装置和通道关系
色彩与材质：
灯光：
相机与画幅：
视角清单：入口广角 / 人眼平视 / 核心展项近景 / 回望节点
文字政策：中文、LOGO、数据、二维码后期添加
避免项：
低质量修复：透视、尺度、人物、文字、材质、曝光和信息屏的处理建议
后期分层：人物 / 内容屏 / 标识文字 / LOGO / 数据 / 光效
待确认项：
```

## 通用提示词骨架

```text
[企业品牌展厅的具体展区]，位于[空间与结构条件]，承担[故事线角色]，
以[空间构成]呈现[核心内容]，使用[色彩与材质]，采用[灯光策略]，
[相机位置、焦段感和画幅]，专业展陈设计，真实尺度，合理通道，
可实施感的结构与收口，概念效果参考；画面内不生成准确中文、LOGO、二维码或企业数据。
```

## 负面约束

根据场景选择：错误透视、尺度失真、堵塞通道、悬浮结构、不可安装构件、过度镜面、无意义大屏、乱码中文、变形 LOGO、虚构企业数据、舞台化灯光、廉价科技感、同质化蓝色光带。

## 交付规则

每个展区分别输出正向提示词、避免项、后期文字与 LOGO 说明、关键假设和待确认项。没有实际调用图像工具时，不声称已经生成图片。

每个关键 `Z##` 至少一张交接卡；视角数量按提案决策需要确定，不机械追求多图。提示词不得增加 `project-manifest.json` 和 `design-model.json` 中不存在的企业事实、设备、尺寸或确定落位。
