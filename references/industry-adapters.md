# 行业适配器

核心模型始终先运行。根据企业正式资料选择最多两个适配器；证据不足时使用 `generic`，不得仅凭企业名称猜测行业。适配器只改变提问、信息顺序、展项候选和风险检查，不增加企业事实。

## `technology-manufacturing`

触发证据：技术研发、产品制造、质量验证、产业应用等资料明确。

- 信息顺序：行业问题 → 原理/技术 → 产品体系 → 制造与质量 → 案例验证 → 研发未来。
- 优先展项：实物、样件、剖切/透明模型、工艺流程、质量对照、数字孪生。
- 关键提问：技术差异如何被非技术访客理解；哪些数据可以公开；实物如何更新维护。
- 反套路：不能用大屏和蓝光代替原理解释，不能把“先进”当证据。

## `group-industrial-park`

触发证据：集团板块、园区、区域发展、产业链、招商或生态伙伴明确。

- 信息顺序：集团定位 → 区域价值 → 产业地图 → 园区能力 → 生态伙伴 → 招商服务 → 未来布局。
- 优先展项：地图、关系网络、产业链、服务旅程、可更新企业名录、多路线导览。
- 关键提问：不同受众如何走不同路线；名录和数据谁维护；集团与子品牌层级如何呈现。
- 反套路：避免把组织架构图放大成空间主叙事，避免用未经确认的招商数字。

## `service-platform`

触发证据：服务流程、平台机制、交付体系、客户案例或网络能力明确。

- 信息顺序：客户问题 → 服务旅程 → 解决方案 → 交付机制 → 信任证明 → 案例 → 服务网络。
- 优先展项：场景推演、流程互动、案例前后对照、服务触点、运营数据快照。
- 关键提问：服务如何可视化；案例的客户授权和数据口径；平台价值如何被证明。
- 反套路：避免抽象“赋能”墙和没有证据的生态口号。

## `culture-heritage`

触发证据：品牌源流、人物、工艺、档案、地域文化或产品演进明确。

- 信息顺序：源流 → 人物与关键选择 → 工艺/精神 → 产品演进 → 当代表达 → 未来传承。
- 优先展项：原件/复制件、档案、工艺演示、口述、声音、触觉、材料样本。
- 关键提问：真实性与版权；文物/档案保护；工艺能否演示；传统如何连接当代业务。
- 反套路：避免符号装饰脱离史实，避免把地域文化当通用纹样。

## `generic`

使用访客决策、四层信息、故事—空间候选、展区/路线/展项和视觉导视通用模型。明确记录无法选择行业适配器的原因和需要补充的证据。

## 组合规则

组合时指定主适配器和辅适配器，例如“科技制造为主、文化传承为辅”。同一内容只保留一条主叙事；出现冲突时以访客决策任务和企业证据优先，不把两个行业目录机械拼接。
