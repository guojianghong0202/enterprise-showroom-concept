# Enterprise Showroom Concept

一个面向企业品牌展厅、企业馆与形象展厅的两阶段概念策划 Skill。

它把品牌资料与空间资料转化为可审阅、可追溯、可继续制作 PPT 的策划成果，并坚持先确认故事线、再展开完整方案。

## 能做什么

- 诊断项目目标、受众、接待场景、必展内容与资料缺口
- 仅使用企业官网、官方公众号、年报、政府或协会等官方来源补充事实
- 生成 2–3 条主题故事线，供用户比较和确认
- 规划展区关系、参观动线、内容大纲与视觉方向
- 生成分区效果图提示词，并可选调用现有的效果图 Skill 深化
- 交付完整策划文档与可直接制作 PPT 的逐页提案脚本
- 通过内置脚本检查输入条件和交付完整性

## 工作方式

### 第一阶段：诊断与故事线

输出：

1. `01_需求诊断与证据台账.md`
2. `02_故事线比选与确认记录.md`

生成 2–3 条故事线后停止，等待用户选择、组合或批准。

### 第二阶段：完整概念方案

故事线确认后输出：

1. `03_企业展厅概念策划母稿.md`
2. `04_PPT逐页提案脚本.md`
3. `05_分区效果图提示词.md`
4. `06_来源与待确认清单.md`

## 推荐输入

- 企业介绍、品牌手册、产品与业务资料、案例、发展历程
- 平面图、尺寸、现场照片、出入口、消防与结构限制
- 项目目标、核心受众、接待场景、预计参观时长
- 企业官网、官方公众号、年报等官方资料入口

资料不足时，Skill 会明确输出缺口、风险或降级方案，不虚构尺寸、面积和工程结论。

## 安装

将仓库克隆到 Codex Skills 目录：

```powershell
git clone https://github.com/guojianghong0202/enterprise-showroom-concept.git "$HOME/.codex/skills/enterprise-showroom-concept"
```

重新打开 Codex 后即可使用。

## 使用示例

```text
请使用 enterprise-showroom-concept，为这家企业策划一个品牌展厅。
先检查我提供的品牌资料、平面图和现场照片，只输出第一阶段的需求诊断和 3 条故事线。
```

用户确认故事线后：

```text
选择故事线 B，并融合故事线 A 的“技术演进”章节。请进入第二阶段，生成完整策划文档和 PPT 逐页脚本。
```

## 目录说明

- `SKILL.md`：核心工作流与触发说明
- `agents/openai.yaml`：Skill 展示配置
- `references/`：证据、叙事、动线、视觉和交付规范
- `assets/`：诊断、策划母稿与 PPT 脚本模板
- `scripts/`：输入与交付验证脚本

## 边界

本 Skill 用于概念策划，不替代建筑、结构、消防、机电、造价或施工图等专业设计工作。
